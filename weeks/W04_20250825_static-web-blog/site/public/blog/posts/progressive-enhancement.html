Progressive Enhancement w/ HTMX + Vanilla JS

Baseline HTML first. Enhancement should be optional, not required.

Baseline content is static HTML

<ul id="articles">
  <li><a href="/blog/posts/semantic-accessibility.html">Semantic HTML</a></li>
</ul>
<noscript>Filtering requires JavaScript, but the list still works.</noscript>

HTMX for progressive swapping (no framework)

<div id="list"
     hx-get="/blog/partials/list.html?tag=css"
     hx-swap="innerHTML"
     hx-trigger="click from:#filter-css"></div>

<button id="filter-css">Filter CSS</button>

If HTMX fails, users still have the baseline list.

Classic JS enrichment (event listeners)

<button id="theme-toggle">Toggle theme</button>
<script>
  document.getElementById('theme-toggle')
    ?.addEventListener('click', () => {
      document.documentElement.classList.toggle('dark');
    });
</script>

Keep it small, feature-scoped, and safe to ignore.

Failure modes you can live with

  • Network down → partials fail: baseline HTML remains usable.
  • JS blocked → no filters/toggles: links still navigate.
  • Slow device → minimal script, no main-thread hogs.
