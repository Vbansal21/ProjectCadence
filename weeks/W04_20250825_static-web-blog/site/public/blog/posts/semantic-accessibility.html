Semantic HTML for A11y & SEO

The fastest “performance optimization” is a DOM that tools understand without JS.

Landmarks first, ARIA last

Prefer native elements before ARIA roles. If you must add roles, never contradict native semantics. 

<header>...</header>
<nav aria-label="Main">...</nav>
<main id="content">...</main>
<aside>...</aside>
<footer>...</footer>

Headings form an outline

One h1 per page sectioning context is fine, but don’t skip levels arbitrarily.

<h1>Title</h1>
<h2>Section</h2>
<h3>Subsection</h3>

Keyboard and skip link

<a href="#content" class="sr-only focus:not-sr-only">Skip to content</a>

Ensure visible focus rings and sequential tab order.

Canonical, description, and structured data

<link rel="canonical" href="https://example.com/blog/semantic-accessibility/" />
<meta name="description" content="..." />
<script type="application/ld+json">{
  "@context":"https://schema.org",
  "@type":"BlogPosting",
  "headline":"Semantic HTML for A11y & SEO",
  "datePublished":"2025-08-25",
  "author":{"@type":"Person","name":"Vaibhav Bansal"}
}</script>

Alt text and media

Describe purpose, not pixels. Decorative images should be alt="" and role="presentation".

Checklist (quick, practical)

  • lang attribute on <html>
  • Landmarks present (header, nav, main, footer)
  • Skip link works
  • Consistent heading outline
  • Descriptive title and meta description
  • Alt text on informative images
  • Valid canonical URL
