Semantic HTML for A11y & SEO

Structure is your first accessibility layer and your first SEO hint. Use the right elements, then add ARIA only where HTML can’t express intent.

Why semantics matter

Semantic HTML gives meaning to structure (<h1>…<h6>, <nav>, <main>), which screen readers, search engines, and browsers can use immediately—no JS required. Start with native semantics; layer ARIA only to fill gaps. That approach yields better keyboard navigation, more robust rendering, and clearer signals for crawlers. :contentReference[oaicite:0]{index=0}

Landmarks before ARIA

Use <header>, <nav>, <main>, <aside>, <footer> to expose page regions. If you must use ARIA, don’t override native semantics; ARIA is a supplement, not a replacement. :contentReference[oaicite:1]{index=1}

<header>...</header>
<nav aria-label="Main">...</nav>
<main id="main">...</main>
<aside aria-label="Related">...</aside>
<footer>...</footer>

Headings as an outline

Headings communicate hierarchy to assistive tech and search. Keep levels logical; avoid skipping for visual styling alone. If you must style, still use proper heading elements or ARIA role="heading" with a level. :contentReference[oaicite:2]{index=2}

Links vs. buttons

  • Link (<a href>) navigates to a URL.
  • Button (<button>) performs an action on the current page.

Don’t attach click handlers to generic <div>/<span> for interactive controls—keyboard users will lose out. :contentReference[oaicite:3]{index=3}

Images, alt, and figures

Use purposeful alt. Decorative images should have empty alt="". Pair diagrams with <figure> + <figcaption>. :contentReference[oaicite:4]{index=4}

Forms with labels

Associate labels with inputs; announce errors via aria-describedby; group related controls with fieldset/legend. :contentReference[oaicite:5]{index=5}

<label for="email">Email</label>
<input id="email" name="email" type="email" aria-describedby="email-hint" />
<p id="email-hint" class="text-neutral-400">We'll never share your email.</p>

Tables with structure

Add a <caption>; use <th scope>; separate <thead>/<tbody>. :contentReference[oaicite:6]{index=6}

<table>
  <caption>Color contrast samples</caption>
  <thead><tr><th scope="col">Text</th><th scope="col">Background</th><th scope="col">Ratio</th></tr></thead>
  <tbody><tr><th scope="row">#e5e5e5</th><td>#0a0a0a</td><td>13.5:1</td></tr></tbody>
</table>

Metadata & structured data

Give each page a specific <title> and meta name="description". Use canonical URLs to avoid duplicates. Add Article/BlogPosting structured data for richer results. :contentReference[oaicite:7]{index=7}

    One-page checklist

    • <html lang="en">, specific <title>, and meaningful <meta name="description">. :contentReference[oaicite:8]{index=8}
    • Landmarks in place; add ARIA only when HTML lacks semantics. :contentReference[oaicite:9]{index=9}
    • Consistent heading outline (or ARIA role="heading" + level if needed). :contentReference[oaicite:10]{index=10}
    • Links navigate; buttons act. No interactive div/span. :contentReference[oaicite:11]{index=11}
    • Alt text conveys purpose; decorative images use empty alt. :contentReference[oaicite:12]{index=12}
    • Forms have labels; errors are announced; groups use fieldset/legend. :contentReference[oaicite:13]{index=13}
    • Tables: caption, header cells with scope, proper sections. :contentReference[oaicite:14]{index=14}
    • Structured data (Article/BlogPosting) for rich results. :contentReference[oaicite:15]{index=15}
